Batch Processing


The Batch Processing module can be used to run the same configuration file on multiple datasets sing the same psfx file without having to go into the configuration for every dataset to change the location of the source files, the output database name, and the location of the binary store.

It is particularly useful for processing datasets from deployments of multiple autonomous recorders, but also has application in reprocessing of any data where you want to run the same PAMGuard algorithms on multiple sets of data.

Currently, it can only be used to process raw data files in normal mode. We are currently developing a version that can run viewer mode tasks on multiple datasets, such as reclassifying clicks, or running the click train detector.

Installation

The batch processing module is a plugin module, which needs to be added to your PAMGuard installation

If you're reading this in the PAMGUard online help, you've already installed the Batch Processing module.

Otherwise, you need to download the latest jar file (e.g. PamguardBatch_1.6.jar) by following links from the PAMGuard plugins pages, then copy the jar file into your plugins folder (e.g. C:\Program Files\Pamguard\plugins) from where PAMGuard will automatically add it to the list of available modules in the Utilities menu next time it starts.

Next: Configuring the batch processor.

Instructions:

1. Make a configuration you're happy with which works on one dataset.

2. Close it and put to one side.

3. Make a new configuration containing just a database and the batch processing module.

4. Near the top, select the configuration file you made earlier

5. Create individual jobs, or a set of jobs (easier when you've a nice folder structure with a sub folder per set of raw data).

6. Set it to run 2 or 3 jobs at a time (depends on your computer) Start at 2 and if the computer isn�t sweating you can increase it to 3 when it�s going. Any more has little gain since it all just slows down. You right click on the list of machines (there is only the one) to do this.

7. Press the normal start button on the batch config and it will launch jobs. For each it will use the same psfx file, but will overwrite the input folder and the binary and database outputs. Note that it only works for folder analysis.
